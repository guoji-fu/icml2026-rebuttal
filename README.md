# Empirical Verification: SWD Scaling for Adaptive Drift SGMs

This repository contains the PyTorch implementation and reproduction instructions for the empirical evaluations presented in our rebuttal. 

To validate our theoretical findings regarding the critical dimension threshold ($d>2$) and demonstrate the efficacy of our proposed Adaptive Drift schedule, we provide the code to reproduce the 10-run statistical evaluations on two severe topological stress-tests.

## 📌 Experiments Included

The provided scripts evaluate the networks' robustness to zero-density vacuums and empirically verify the structural breakdown for $d>2$ across two geometric stress-tests: 
1. **Concentric Hyperspheres ($d=4, D=40$):** Data is uniformly sampled from two disjoint manifolds (radii 1 and 4) embedded via random orthogonal rotation, creating a massive radial vacuum.
2. **Subspace Mixture of Gaussians ($d=4, D=40$):** 8 well-separated modes with tight variances, testing the network's ability to cross deep, low-density valleys.

Both experiments compare the **Standard OU** schedule against our **Adaptive Drift** schedule across dataset sizes $n \in [100, 10000]$.

## ⚙️ Requirements

The code requires a standard PyTorch environment. The primary dependencies are:
* `torch`
* `numpy`
* `matplotlib`
* `tqdm`

## 🚀 How to Run

We have consolidated the training loops, SDE schedules, and Sliced Wasserstein Distance (SWD) evaluation into a single, easily executable script.

To execute the 10-run statistical evaluation for both environments, simply run:

```bash
python exp_sphere.py
python exp_mog.py
```

(Note: The script will automatically detect and utilize a CUDA-enabled GPU if available. Due to the 10 independent runs for statistical significance, the full execution may take some time depending on your hardware.)
